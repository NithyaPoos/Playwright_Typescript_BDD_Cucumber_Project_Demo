Cucumber is behavior‑driven and 
it was created for non‑technical people (product owners, QA, business analysts) to read and write tests.
The feature file describes behavior in plain English using gherkin syntax(Given when then)
The step definitions contain the actual code
Cucumber matches the English text to the step code
Playwright runs inside the step definitions

Run:
npx bddgen
npx cucumber-js --dry-run
npx cucumber-js
or npm test
Cucumber will:
Read all .feature files
Find matching step definitions
Execute the Playwright code inside them

Feature file (.feature)	-Defines scenarios or describe behavior  in English
Step definitions (.steps.ts)	-	Contains Playwright code(implement behavior)
Playwright	-Automates the browser(Playwright runs inside those step definitions)

const loginPage
→ A local variable
→ Exists only inside that one step
→ Disappears immediately

this.loginPage
→ A shared variable
→ Lives for the entire scenario
→ Accessible in all steps
→ Perfect for Page Objects
Use this.loginPage when:
You need the page object in multiple steps
You want to store state across steps
You want consistent behavior

Use const loginPage when:
Everything happens inside one step
You don’t need to reuse the object
